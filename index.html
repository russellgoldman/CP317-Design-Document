1. Introduction


BeerBuddy will be a beer utility application that is web based. This application aims to be a useful tool to try out new kinds of beer regardless of beer expertise. Users will be presented an introductory page which will contain two modes: one for novice users and the second one for experienced users. After suitable search criteria is filled out, users will then be provided with beer recommendations catered to their individual knowledge and taste.


2. Object Classification


2.1 Entity Objects


  • Brand

    A table to hold individual brand information (name, country of origin)

  • Beer

    A table to hold individual beer information and foreign key relationships (info: name, colour, alcohol percentage; relation: brand, body type, container types)

  • BodyType

    A table to hold body type information (light, medium, dark)

  • Rating

    A table to hold rating information for individual beers (rating value from 1-5, rater name, date)

  • ContainerType

    A table to hold container type information (can, bottle, keg)

  • Taste

    A table to hold taste information (aggressive, complex, fruity, hoppy, malty, crisp)


2.2 Boundary Objects


  • BrandView

    Where a user views information relevant to a Brand

  • BeerView

    A table to hold individual beer information and foreign key relationships (info: name, colour, alcohol percentage; relation: brand, body type, container types)

  • RatingView

    Where a user views information relevant to a Rating

  • SearchBeerView

    Where a user searches for a Beer

  • SearchBeerResultsView

    Where a user sees the results of a search for a Beer

  • CreateBrandView

    Where a user fills out and submits a form required to create a Brand

  • CreateBeerView

    Where a user fills out and submits a form required to create a Beer

  • RateBeerView

    Where a user fills out and submits a form required to rate a Beer (create a Rating)


2.3 Control Objects


  • BrandController

    Coordinates all activities related to Brand objects

  • BeerController

    Coordinates all activities related to Beer objects

  • RatingController

    Coordinates all activities related to Rating objects

  • SearchController

    Coordinates all activities related to searching for beers


3. Use Cases


3.1 Search


  • SearchBeerView is created and takes input from user

  • SearchController uses input to access the Beer entity

  • SearchController finds all Beer objects with matching input to user input and creates a data structure with all the objects

  • SearchController creates a SearchBeerResultsView with the list of Beer objects

  • SearchBeerView and SearchBeerResultsView are destroyed after use case ends


3.2 Review


  • CreateRatingView is created and takes input from user

  • RatingController creates a new Rating object with information from CreateRatingView

  • RatingController attaches the Rating object to the corresponding Beer object

  • (System to determine if rating is successfully added to database TBD)

  • CreateRatingView is destroyed after use case ends

  • New RatingView can be created by RatingController pertaining to new Rating object


3.3 Enter Beer Data


  • CreateBeerView is created and takes input from user

  • BeerController creates a new Beer object with information from CreateBeerView

  • (System to determine if beer is successfully added to database TBD)

  • CreateBeerView is destroyed after use case ends

  • New BeerView can be created by BeerController pertaining to new Beer object


3.4 Enter Brand Data


  • CreateBrandView is created and takes input from user

  • BrandController creates a new Brand object with information from CreateBrandView

  • (System to determine if brand is successfully added to database TBD)

  • CreateBrandView is destroyed after use case ends

  • New BrandView can be created by BrandController pertaining to new Brand object
